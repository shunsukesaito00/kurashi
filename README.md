# くらしの計算室

日常でよく検索される計算(手取り・消費税・和暦変換など)をブラウザだけで完結させる、
日本語の無料ツールサイトです。収益化は商用クエリ向けアフィリエイトを主軸とし、
Google AdSense は副次的な収益源として位置づけています。

## サイト構成

| ページ | 内容 |
|---|---|
| `index.html` | トップ(ツール一覧) |
| `tools/tedori.html` | 手取り計算(月収→手取り) |
| `tools/zeikomi.html` | 消費税計算(税込⇔税抜) |
| `tools/wareki.html` | 西暦・和暦変換 |
| `tools/nenrei.html` | 年齢計算 |
| `tools/bmi.html` | BMI・適正体重 |
| `tools/tsumitate.html` | 積立・複利シミュレーション |
| `tools/waribiki.html` | 割引(〇%オフ)計算 |
| `tools/jikyu.html` | 時給換算 |
| `tools/moji.html` | 文字数カウント |
| `privacy.html` | プライバシーポリシー(AdSense審査に必須)。`noindex` であり `sitemap.xml` には載せない |
| `about.html` | 運営者情報(AdSense審査に有利) |

フレームワーク・ビルド不要の静的HTML/CSS/JSのみ。サーバーサイド処理はありません。

## 公開までの手順(必須)

1. **ホスティング**: 完了済み。GitHub Pages で配信中です。
   公開URL: https://shunsukesaito00.github.io/kurashi/
   `main` へのプッシュ後は `git push origin main:gh-pages` で本番を同期する。
2. **独自ドメイン取得(強く推奨)**: AdSense はサブドメイン(`*.github.io` 等)では
   審査に通りにくいため、独自ドメイン(年1,000〜1,500円程度)を取得して割り当てる。
3. **残りのプレースホルダ置換**:
   - `about.html` の運営者名・連絡先(`準備中` 表記)を実際の情報に書き換える
   - 独自ドメイン取得後: `sitemap.xml` と `robots.txt` の
     `https://shunsukesaito00.github.io/kurashi/` を実ドメインに置換する
     (現在は github.io URL 済み。`YOUR-DOMAIN.example` は残っていない)
4. **Google Search Console に登録**し、`sitemap.xml` を送信する。

## 収益化ロードマップ(月5万円へ)— 2026年7月改訂版

**改訂の理由**: フルAI生成コンテンツを前提に精査した結果、旧ロードマップの主軸
「情報検索SEO → AdSense」は2026年の環境では成立しないと判断した。

- AI Overviews の浸透で情報クエリのゼロクリック化が進行
  (Google検索の流入率は2025年に41.1%まで低下、AIO表示クエリの83%がゼロクリック)。
  本サイトが狙う「手取り 計算」「令和8年 西暦」はまさにAIが直接回答する領域。
- 完全AI生成コンテンツのみでの AdSense 審査通過は2026年時点で実質不可能との報告が多数。
- 旧フェーズ2の「ロングテール記事のAI量産」は、Googleスパムポリシーの
  「大量生成コンテンツの不正使用」に該当するリスクがあるため**撤回**。
- 一方、商品比較・申込などの**商用クエリの流入は残っている**(流入率62.3%)。
  また **KDP(Kindle出版)はAI生成を申告すれば公式に許可**されており、
  集客がGoogle検索に依存しない販売チャネルとして有効。

この前提で、収益の主軸を「商用クエリのアフィリエイト + 検索非依存チャネル」に置く。

### フェーズ1: 商用導線への転換(〜2か月)

- A8.net・もしもアフィリエイト等のASPに登録し(**運営者にしかできない作業**)、
  商用クエリに接続できるツールへ成果報酬型の導線を実装する:
  - 積立シミュレーション → 証券口座開設(1件5,000〜15,000円)
  - 時給・手取り計算 → 転職サービス(1件3,000〜15,000円)
- 月5万円 = **月5〜10成約**。PV量ではなく成約率で勝負する。
- AdSense は「申請はするが主軸にしない」に格下げ。審査には運営者情報の充実など
  人間の関与が実質必須のため、申請前に `about.html` を実名義で埋めること。

### フェーズ2: 検索非依存の販売チャネル(2〜4か月)

- **Kindle出版(KDP)**: AI生成コンテンツはKDP管理画面で申告すれば規約上問題なし
  (申告は購入者には非表示・検索順位にも影響なし)。1日3タイトル上限・品質審査あり。
  実用系(計算・手続き・テンプレート集)を月2〜4冊のペースで刊行し、
  Amazon内検索という別の集客経路を持つ。
- **デジタル商品販売(BOOTH・note)**: スプレッドシートテンプレート等を
  少額(500〜1,500円)で販売。マーケットプレイス内の検索・回遊が集客源。

### フェーズ3: AIが代替できない機能への投資(継続)

- 記事の量産はしない。代わりに「AIが答えを言えても代替できない」機能を強化する:
  複数条件の比較シミュレーション、PDF/印刷出力、結果の保存・共有など。
- ツールの追加は検証済みの計算ロジックを伴うものに限る(例: 育休給付金計算、
  退職金の税金計算)。1本ごとに一次情報(公的機関)との突合を行う。

### 正直な見通し

フルAI生成で「確実に」月5万円へ到達する経路は2026年時点で存在しない。
本ロードマップは期待値が最も高い構成への組み替えであり、成否は
コンテンツ生成量ではなく「商用導線の成約」と「販売チャネルの分散」で決まる。
ASP・KDP・決済のアカウント開設は運営者本人にしかできず、ここが実行の律速になる。
